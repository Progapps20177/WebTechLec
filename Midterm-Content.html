CSS

Cascading Style Sheets (CSS) - used to specify presentation of HTML elements or present the design of a web page. E.g. layout, formatting, fonts, colors, etc.

- It was developed by Hakon Wium Lie and Bert Bos.

Versions of CSS

CSS Level 1 - the first version of CSS was released on 1996. The specification of CSS 1 at that time was for designing text, images, and aligning for the elements.

CSS Level 2 - new features were added to CSS such as aural style sheets, use of media types and positioning for HTML elements.

CSS Level 2 Revision - this version focused on fixing the errors of CSS Level 2. Some fixes include removing useless features and adding browser extensions specifications.

CSS Level 3 - features in this version were called modules. Each module could add new features or extend the capabilities of CSS Level 2.

CSS Level 4 - the features of CSS 4 were split into level 4 modules.

HTML Stylesheets

- sources of style for HTML documents.

1. Author Styles - specifically connected to the HTML itself

2. User Styles - can specify a user's preferred style information for a particular document. Very useful for any web user especially people with disabilities.

3. User Agent Styles - the default style of a browser.

4. Inline Styles - it is used for directly putting a CSS style for a specific type of HTML element.

5. Embedded Stylesheets - can define styles for a whole HTML document in one place. Simply add < script > in the head of an HTML document.

6. External Stylesheets - used to define a style for multiple HTML documents. The stylesheet is linked in the HTML file by using < link >.

CSS Statements

- follows the syntax of CSS to ensure readability.

- statements have two kinds: At-rules and Rulesets

Basic Building Blocks are:

1. Property - the CSS identifier; defines what feature is used.

2. Value - describes how the feature is used in the browser.

CSS Declarations - it pairs the property and value that apply to all CSS elements.

- Uses a colon to distinguish the property and the value.

- Can use blocks to group the declarations.

At-rules

- a statement that provides instructions on how CSS will perform or behave. Always starts with an @(at) sign. Examples of at-rules:

1. @charset - defines the character set.

2. @import - requests and includes an external CSS file.

3. @namespace - useful for applying CSS to XML HTML.

4. @media - targets styles to specific screens.

5. @supports - tests whether a feature is supported by the browser.

6. @page - defines styles for individual pages when printing the document.

7. @font-face - load custom fonts on a webpage.

8. @keyframes - does animation.

Rulesets

- relates a collection of CSS declarations to a condition depicted by a selector.

- consists of a selector, followed by a brace-enclosed declaration block. The declaration block contains one or more declarations separated by semicolons.


> CSS Selector - patterns that are used to select the HTML elements you want to style.

> CSS Multinavigational - it provides for responsive and touch-friendly behavior in a website.

> nav - it shows in the section of the webpage providing links from a document to other documents.


CSS Position Property

1. Fixed - the position is the same with the browser window.

2. Absolute - the elemet is positioned relative to its predecessor element.

3. Relative - element is in a normal position.


> nth-child() Selector - every element that would match the number of children based on the group of siblings.

> hover Selector - used for pointing the element the user interacts with but it doesn't initiate it but rather it triggers over the element.

> target Selector - used for the styling of the current active element.


The Laws of UX

- a collection of principles that adhere to a good design.

1. Fitts's Law - make elements have a large size and easy to select.

2. Hick's Law - divide long questions to be easily answered by users.

3. Jakob's Law - provide familiar design patterns.

4. Law of Prägnanz - complex shapes must have simplicity and order.

5. Law of Proximity - clusters of content are always grouped together.

6. Law of Similarity - text elements must be visually different from links and navigation systems.

7. Law of Uniform Connectedness - similar functions should be grouped through colors, shapes, etc.

8. Miller's Law - content is organized in groups of 5-9 items at a time.

9. Occam's Razor - remove unnecessary elements while retaining the the overall function.

10. Serial Position Effect - place the most important elements at the first and last.


Asterisk (*) - the universal selector.

CSS Preprocessors - generates CSS using a custom language symbol that includes features that doesn't exist in pure CSS.

CSS Frameworks - provide predefined CSS design functionality that can be reused, extended, and customized, i.e. bootstrap

CSS Polyfills - provides features that a developer expects a browser to have natively.


Javascript

- the programming language of HTML. It should be at the beginning of an HTML document to be prioritized more with the other javascript code.

- Core Javascript (Ecmascript) language and API

- (Traditional) Browser Object Model (BOM) API (non-standardized)

--- Window, Navigator, Screen, Location, History


Server side Scripting - used for web browsers to produce a response for each client or user's request to the website.

Client side Scripting - runs the script in a browser. The source code from a web server is transferred to a user's computer then run in the browser.

Top-level code - executes as the script is loading.

Document Object Model (DOM) API

- Node, Document, Element, Text, Attr, DocumentType, Comment, etc/

- Event, EventTarget, EventListener, etc.

- CSSStylesheet, CSSRule, etc.

> Node - primary type of DOM.


Miscellaneous Javascript Web APIs

Geolocation, IndexedDB, Local Storage, Push Notifications, Service Workers, Web Sockets, Web workers, XML HTTP Request (AJAX), etc.

- most of the APIS are accessed from scripts via the global object window.

Document Object Model

- API that allows access to the HTML document from within scripts associated with the web page.

- a parsed HTML document is represented by a DOM tree, which contains nodes representing elements, element attributes, textual content, and other HTML document components (e.g. DOCTYPE, comments, etc.).

- accessed via the Document object property of the global window object.

Functionalities:

1.) Retrieve references to nodes (or nodes collection)

a. getElementByID()

b. getElementsByTagName(), getElementsByClassName()

c. querySelector(), querySelectorAll()


2.) Traverse the DOM tree (from a given node)

a. parentNode, parentElement

b. childNodes, children


*Whitespace is considered as text in making an HTML page.
