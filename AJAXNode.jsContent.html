AJAX and Node.js Notes

AJAX

Description: AJAX is an acronym for Asynchronous JavaScript and XML. It can send and receive information in different data formats, such as HTML, JSON and others. Its best characteristic is being able to update a page without having to reload it. For server communication, AJAX makes use of the XMLHTTPRequest object.

4 Major Features:

  • Updates a page without reloading it.
  • Request or receive data from a server after loading a page.
  • Send data to a server while in the background.

How AJAX works:

The browser loads the AJAX engine instead of a traditional web page. When the page has loaded, the engine continues running in the background while using JavaScript to communicate with the browser. Whenever the user interacts with the page like clicking on the page, a JavaScript call is sent to the AJAX engine so it is prepared to respond instantly. If additional data is required then the engine sends a request to the server through the XML while also updating the page.

AJAX functions and methods

AJAX event methods

.ajaxComplete() - registers a handler to be called when AJAX requests complete.

.ajaxError() - registers a handler to be called when AJAX requests complete with an error.

.ajaxSend() - attaches a function to be executed before an AJAX request is sent.

.ajaxStart() - registers a handler to be called when the first AJAX request begins.

.ajaxStop() - registers a handler to be called when all AJAX requests have completed.

.ajaxSuccess() - attaches a function to be executed whenever an AJAX request completes successfully.


jQuery.ajax() - performs an asynchronoues HTTP (AJAX) request.

jQuery.ajaxPrefilter() - handles custom AJAX options or modify existing options before each request is sent and before they are processed by $.ajax().

jQuery.ajaxSetup() - sets default values for future AJAX requests. Not recommended for use.

jQuery.ajaxTransport() - creates an object that handles the actual transmission of AJAX data.


Node.js

Description: Node.js is a JavaScript run-time environment which uses an event-driven, non-blocking I/O model that makes it light and effective. It is built on Chrome's V8 JavaScript engine. A non-blocking I/O basically means that multiple data requests can be initiated in parallel or independently.
